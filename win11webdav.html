以下內容按「內建功能 → 免安裝指令 → 第三方工具 → 進階 CLI」的難度與彈性遞增排序，說明在 Windows 11 掛載 WebDAV 的最佳做法、常見地雷與優化技巧。可依自己的需求（偶爾存取、長期掛載、需進階快取／加密…）挑選。

1  用檔案總管＋「WebClient」服務（最無痛，桌面使用者友善）

  1. 啟用 WebClient 服務

    • Win + R → services.msc，找到 WebClient，啟動類型設為 Automatic (Delayed Start) 後按「啟動」。
      若服務沒啟動，任何 WebDAV 映射都會失敗。servicecenter.fsu.edu

  2. 解除 50 MB 傳輸上限（可選，但幾乎必做）

    • regedit → HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\WebClient\Parameters

    • 將 FileSizeLimitInBytes 改為 4294967295 (十進位)，對應 4 GB。重啟 WebClient 或電腦生效。wintips.orgtech2geek.net

  3. （若伺服器僅支援 Basic Auth 或 HTTP）允許基礎驗證

    • 同一機碼下把 BasicAuthLevel 設為 2 後重啟。值 2 代表允許 HTTP/HTTPS 的 Basic/Digest Auth。help.dreamhost.comdocs.qnap.com

    安全提示：僅在私有網或 VPN 中使用 HTTP/Basic Auth；對外公開務必走 HTTPS。

  4. 映射方式（兩選一）

    • 檔案總管 → … →〈對應網路磁碟機〉

      • 資料夾輸入：https://example.com/webdav/

      • 打勾「重新登入時還原連線」。

    • 「新增網路位置」精靈（部分環境對 HTTPS 較穩定）。
      thewindowsclub.comumatechnology.org

  5. 命令列自動化

    powershell
    net use Z: "https://example.com/webdav" /user:帳號 密碼 /persistent:yes

    透過登入指令碼或工作排程在開機時掛載。learn.microsoft.com

適用情境：偶爾編輯檔案、檔案 < 4 GB、不需進階快取；優勢是內建、無安裝。
局限：連線掉線時易卡住，對大量小檔快取效能不佳，缺乏檔案鎖定提示。

2  第三方掛載器（免折騰、功能完整）

工具授權與費用特點何時推薦
RaiDrive 2025.7個人免費、商業收費圖形化、可同時掛多個雲端／WebDAV、支援自動重新連線與快取想要「像隨身碟一樣」操作且不想改 Registry
Mountain Duck 5付費以 Finder/Explorer 介面加上背景同步與加密資料夾需要離線緩衝、端對端加密、商業協作
Cyberduck (只掛載時建議搭配 Mountain Duck)開源單次上傳下載方便、支援 WebDAV 版本控制管理檔案、偶爾大量傳輸
NetDrive 4付費與 RaiDrive 類似，介面較商用，支援磁碟快取企業授權或需要專線技術支援

特色：不受 Windows 50 MB 限制、掉線會自動重連、支援檔案上傳續傳與檔案鎖。
建議仍使用 HTTPS，並在軟體內開啟「重試 / 自動重新連線」。

3  rclone mount ＋ WinFsp（CLI 玩家／自動化最佳）

  1. 安裝 WinFsp，再安裝 rclone。WinFsp 提供 FUSE 兼容層，rclone 才能在 Windows 建立使用者態檔案系統。rclone.org

  2. 建立 webdav 遠端：rclone config → New remote → webdav → 輸入 URL、帳密、Vendor=Other。

  3. 掛載範例：

    powershell
    rclone mount mydav: X: --vfs-cache-mode writes --network-mode

    加 --vfs-cache-mode full 可取得更類似本地磁碟的體驗。

  4. 可放入 NSSM / 任務排程，開機自動掛載。

優勢：

  • 可調整並行、快取大小、檔名編碼、分段上傳等進階參數

  • 同時支援 S3、Google Drive… 統一腳本
    劣勢：需命令列經驗，GUI 使用者門檻高。

4  常見錯誤與排除

錯誤碼／現象可能原因解決辦法
0x800700DF：檔案大小超出限制未改 FileSizeLimitInBytes參考上文第 1‑2 步修改 Registry。wintips.org
0x80070043：網路名稱找不到URL、連接埠或 SSL 設定錯誤；WebClient 未啟動確認 https://、正確 Port；檢查 WebClient 服務。synoforum.com
憑證警告 / 要求安裝憑證伺服器自簽或過期憑證匯入根憑證到「信任的根憑證機構」，或改用合法 CA。
掛載後資料夾空白伺服器僅 Digest/Basic Auth；BasicAuthLevel 未調整設 BasicAuthLevel=2 並重啟。help.dreamhost.com

5  效能與安全最佳化

  • TLS 1.2/1.3：Windows 11 24H2 預設已開啟，伺服器亦需相容，以免握手失敗。learn.microsoft.com

  • 停用 Windows Search 索引：對網路磁碟點右鍵→〈屬性〉→取消「允許檔案內容被索引」；降低大量小檔清單延遲。

  • 不在公網直接暴露 WebDAV：建議放在 VPN／ZeroTier／Tailscale 後，或加 WAF＋限速。

  • 權限分層：避免用管理員帳號掛載，可用最小權限帳號；需要多工作業可各建一個虛擬路徑。

結論：哪一種「最好」？

使用情境建議方案
偶爾需要拿檔案、無權限安裝軟體內建 WebClient + 檔案總管（最快上手）
常態掛載、希望穩定且 GUIRaiDrive（個人免費）或 Mountain Duck（需離線快取／加密）
DevOps、自動備份、批次腳本rclone mount + WinFsp（高度可程式化）
單次大量傳輸或管理版本Cyberduck（傳輸工具而非長期掛載）

依需求權衡：若你只想「按兩下就有磁碟機」，RaiDrive 是目前 Windows 11 上體驗與穩定度綜合最佳的第三方；若追求零安裝、公司電腦無權限，那就用內建功能並把 Registry 調整好；需要自動化或跨雲整合，則 rclone 最有彈性。

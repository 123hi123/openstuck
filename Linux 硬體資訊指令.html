1. CPU 處理器

指令用途／常見旗標
lscpu標準 CPU 概覽（核心數、架構、時脈等）

2. 記憶體 (RAM / Swap)

小提示

  1. 搭配 watch
    想持續監看，把任何指令前面加 watch -n 2（每 2 秒刷新），例如

    bash
    watch -n 2 cat /proc/interrupts

  2. 善用 grep / egrep
    篩選特定欄位或關鍵字，例如 lscpu | grep -E 'Model|Socket'。

  3. 安裝工具包

    • Debian/Ubuntu：sudo apt install sysstat smartmontools lm-sensors inxi lshw hdparm

    • RHEL/CentOS/Fedora：sudo dnf install sysstat smartmontools lm_sensors inxi lshw hdparm

    • openSUSE：sudo zypper install sysstat smartmontools sensors inxi hwinfo lshw

  4. /proc 與 /sys
    幾乎所有硬體資訊都能在這兩個虛擬檔案系統找到，指令就是幫你整理 / 格式化它們。

下面整理 MSI Cubi N ADL（或其他 MSI 桌機／迷你機）「回復原廠設定」的四種做法，按照「是否仍保有原廠 Windows 復原分割區」由簡到難排列。操作前請務必先備份資料並接好電源。

0. 檢查你還有沒有 MSI 復原分割區

  1. 開 磁碟管理（Win + X → 磁碟管理），若看到一個 10 GB↑、沒有磁碟代號、標示 RECOVERY 的 NTFS 分割區，就表示復原映像仍在。

  2. 若已改裝 Linux／重新分割磁碟，看不到該分割區，代表只能用 USB 還原碟或重新安裝 OS（見方法 ③）。

1️⃣ F3 Recovery —— 最完整的「恢復 MSI 工廠映像」

前提：硬碟上仍保留 MSI 原廠復原分割區

步驟說明
1開機看到 MSI logo 時，連續按 F3 進入 Windows RE 介面
2依序選 Troubleshoot → Restore MSI factory settings
3進入 System Partition Recovery，照畫面指示完成重灌

⚠️ 注意：此流程會格式化系統磁碟，過程中請勿斷電。

2️⃣ Windows 11/10「重設此電腦」—— 沒有 F3 分割區也可用

前提：電腦仍能進入 Windows

  1. 設定 → 系統 → 復原（Recovery）→ 重設此電腦。

  2. 選擇

    • 保留我的檔案（僅移除程式與設定）

    • 移除所有項目（整碟清空，較接近出廠）

  3. 再選 雲端下載 或 本機重新安裝，確認後按 重設。support.microsoft.com

這個方法不能還原 MSI 隨機附帶的工具程式與驅動，但能快速得到「乾淨的 Windows」。

3️⃣ 使用 MSI BurnRecovery / 官方 USB 安裝媒體

前提：已刪掉復原分割區、換 SSD，或系統無法開機

  1. 另一台可用的 Windows PC 上下載 MSI BurnRecovery，輸入序號後建立 USB 復原碟。

  2. 在 Cubi N 上插入復原碟 → 開機狂按 F11 叫出 Boot Menu → 選 USB 開機。

  3. 進入 BurnRecovery 介面後，選 Restore MSI factory settings。

  4. 若沒有備份映像，可改用 微軟官方 Media Creation Tool 製作純淨 Windows 安裝碟，再到 MSI 官網抓驅動。tw.msi.com

4️⃣ 恢復 BIOS/UEFI 為出廠值（可搭配前述方法）

前提：想連 BIOS 也一起歸零，或系統設定錯誤導致無法開機

  1. 開機狂按 Del 進 BIOS。

  2. 至 Save & Exit / 儲存及離開 → 選 Load Optimized Defaults / 載入最佳化預設值 → Yes。

  3. 按 F10 存檔離開，系統會重開機並套用預設。tw.msi.com

若因超頻或錯誤設定導致完全無畫面，可 (1) 斷電、拔電池跳線 CLRTC 或 (2) 清 CMOS，步驟請依主機板說明書操作。

② 你（或前一位使用者／店家）曾手動把 Windows 抹掉改裝 Ubuntu系統磁碟只剩 1 或 2 個 Linux 分割區；Disk Management 沒看到 RECOVERY一旦重新分割磁碟、覆寫安裝 Ubuntu，最常見就是把 MSI 的 F3 回復分割區也一併刪掉。所以按 F3 不會進復原精靈，只能用 USB 安裝媒體重裝 Windows。
③ 更換過 SSD（或加裝第二顆 SSD）NVMe 裸裝新碟或換更大容量後才裝的 Ubuntu；原廠 SSD 被你拿去當資料碟/備份原本的恢復映像在舊 SSD；如果你把舊碟拔走或格式化，就等於「失去工廠映像」。可將原 SSD 裝回、改 BIOS 開舊碟啟動，或下載 MSI BurnRecovery 映像製作 USB 再灌回新碟。
④ 賣場販售「Linux 版特仕機」機殼貼紙、外包裝直接寫「Ubuntu Certified」、「N200/8G/256G Linux」少數系統整合商（SI）會批量採用 Ubuntu LTS 做軟路由、工業控制；他們拿到 MSI 原廠裸機後就改刷自己的映像並附 Linux 保固。若想改 Windows，需要自行安裝並處理驅動。
⑤ Windows 開機毀損，Ubuntu 復原環境自動啟動（較少見）原本雙開；Windows 更新失敗後剩下 Ubuntu 能進若之前做過雙系統，Windows 分區壞掉就只剩 Ubuntu；修好 Windows 或重灌即可。

我應該怎麼辦？

  1. 確認自己想用哪個系統

    • 若你已經習慣 Ubuntu 或把它拿來作伺服器，其實不必「回原廠」。

    • 想要 Windows（含原廠驅動、MSI Center 等）才需要重灌。

  2. 如果要回 Windows，又沒有 F3 復原分割區：

    1. 到 MSI 官網 › 支援 › Cubi N ADL 頁面下載驅動備好。

    2. 用另一台電腦執行 Microsoft Media Creation Tool 做一隻 Windows 11 USB。

    3. 在 Cubi N 開機狂按 F11 → 選 USB 啟動 → 全新安裝 Windows。

    4. 進 Windows 後安裝晶片組、網路、音效等驅動（或用 MSI Center 讓它自動抓）。

    5. 輸入合法序號 (COA)；若買的是含 Win 機種，UEFI 內建授權會自動啟用。

  3. 如果你保留了原廠復原分割區（Disk Management 看得到 RECOVERY）：

    • 開機時連續按 F3 → Restore MSI factory settings，就能一鍵重灌回出廠 Windows。

  4. 備份資料
    無論選哪條路，先把 Ubuntu 裡的重要資料備份到外接硬碟或雲端，再動手以免遺失。

小結

  • 你的 Cubi N 之所以是 Ubuntu，多半因為買到「空機版」或後來自行改裝。

  • 仍想用 Ubuntu：保持現狀即可；MSI 官網也有 Linux 驅動或你可沿用社群驅動。

  • 想要回 Windows：若還有復原分割區就按 F3，沒有就用 Win USB 乾淨重灌，再裝驅動或用 MSI BurnRecovery 映像。

照以上判斷與步驟，就能理解「為什麼是 Ubuntu」並依需求切換或還原系統。祝操作順利！

以下整理三種在 Linux (Ubuntu) 圈子裡最常見、兼顧「整機鏡像備份 → 一鍵還原到另一顆硬碟甚至新硬體」的做法。難易度由淺入深；你可依自己習慣與可接受的維護成本挑一種實作。

方案 A ── Clonezilla 離線磁碟映像

最接近「ghost 一鍵備份／還原」的體驗，亦是社群使用率最高的做法。

1. 建立備份

  1. 準備

    • 外接 HDD/SSD (容量 ≥ 目前系統已用空間＋20 %)。

    • 另一支可開機的 USB，燒錄 Clonezilla Live（或直接用 Ventoy 裝 ISO 即可）。

  2. 開機進 Clonezilla → 選 device‑image → local_dev。

  3. 按提示把外接硬碟掛上，選「savedisk」(整顆) 或「saveparts」(單分割區)。

  4. 勾 gzip + sfs 壓縮格式，完成後外接碟裡會生成一個目錄（日期＋機器名）。

2. 還原到新硬體

  1. 把新硬碟裝到新機 → 同樣 USB 開機 → Clonezilla→ restore‑disk。

  2. 指定剛才的映像目錄，選目標磁碟；完成後重開機即可。

  3. 可能需要的收尾

    • 驅動：Ubuntu 核心自帶多數驅動，換新硬體通常能直接開機。

    • grub 修復：若分割表類型改變 (e.g. SATA→NVMe)，可開 liveUSB 執行

      bash
      sudo grub-install /dev/nvme0n1  # 視實際裝置調整
sudo update-grub

    • fstab：Ubuntu 預設用 UUID，通常不用改；若你手動寫 /dev/sdX，記得改成新 UUID。

優點：全系統位元級備份，還原速度快，一鍵化程度高。
缺點：

  • 舊碟 → 新碟必須「新碟容量 ≥ 舊碟已用空間」；否則需先用 GParted 把舊系統分割縮小到足以容納。

  • 重新還原後磁碟分割與舊碟一樣大，若新碟更大需用 GParted 手動拉大分割區。

方案 B ── Timeshift + rsync / Btrfs snapshot

適合你的根分割區已是 Btrfs（或願意改用 Btrfs）；好處是增量快照、省空間，還能跨機還原。

1. 配置

bash
sudo apt install timeshift
sudo timeshift --create --btrfs --comments "pre-upgrade"

  • 若用 ext4，也可改 --rsync。

  • 建議把 /home 獨立分割 & 掛到外接硬碟：

    bash
    sudo timeshift --create --tags D --rsync --snapshot-device /dev/sdb1

2. 還原

  1. 在新硬體安裝「同版／較新版 Ubuntu」。

  2. 安裝 Timeshift → 外接含快照的硬碟 → timeshift --restore，選對快照即可。

  3. 重裝開機管理程式 grub-install（如方案 A 收尾步驟）。

優點：

  • 每日或每次套件升級自動增量快照；快到「打快照 ≈ 打 git commit」。

  • 可隨時滾回舊快照；跨機還原時能直接用新硬體的分割表大小。
    缺點：第一次複製到新硬體仍須做一個乾淨系統安裝（花 10 分鐘），之後再套 Timeshift 還原，沒有 Clonezilla 那種「立即 bit‑copy」。

方案 C ── dd 或 partclone 全碟複製

最原始卻也最萬用的 Linux 工具鏈；只要能讀寫 block device 就能執行。

bash
# 在 liveUSB 下：舊→新
sudo dd if=/dev/sda of=/dev/nvme0n1 bs=64M status=progress conv=noerror,sync

  • 用 pv 或 dcfldd 可看進度；partclone（Clonezilla 內核）也行。

  • 完成後用 gdisk 或 growpart 把分割表大小調整成新碟容量。

優點：不依賴任何第三方工具；跨 OS 通吃。
缺點：

  • 必須 新碟大小 ≥ 舊碟整體容量（非已用空間）；太浪費。

  • 不會壓縮，備份檔動輒數百 GB。

  • 操作失誤風險高（if / of 指錯就全毀）。

小技巧 & 注意事項

  1. 提前縮分割：若未來目標硬碟可能比現在小，請用 GParted 把 root  及 home 分割在 備份前 就縮到 ≤ 目標容量。

  2. GRUB/Fstab 用 UUID：Ubuntu 安裝程式已預設如此；確認 /etc/fstab、/etc/default/grub 沒硬寫 /dev/sdX。

  3. UEFI vs Legacy：新硬體若 BIOS 只開 UEFI，就別把舊系統做成 Legacy MBR；優先 GPT+UEFI。

  4. 網路傳輸備份：Clonezilla 支援 sshfs/NFS/Samba；Timeshift 可配合 rsync --link-dest。對沒有大外接碟的人很方便。

  5. 自動化腳本：Clonezilla 還原流程可以在 ocs-sr -e1 auto 形式寫成 shell script，放在 USB 裡；插碟→選腳本→全自動。

只要先想清楚「還原時要多省事、備份檔要多精簡」，就能從以上三種作法找到適合自己的一鍵備份／還原方案。
多數人第一次嘗試都會用 Clonezilla 完整鏡像（最快最直觀），之後再進階到 Timeshift 或 Btrfs send/receive 做持續快照。祝你備份順利、永不怕重灌！
